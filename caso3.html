Usted como parte de una empresa de software con clientes variados, estos clientes por lo general son empresas bastante grandes, el sistema que ofrecen a los clientes es una aplicación de escritorio donde los empleados de cada empresa registran un resumen de sus funciones durante el día, el sistema es bastante fácil de usar, y ayuda a los supervisores a controlar el trabajo y llegar a los objetivos de la empresa propuestos semanalmente.

Los gerentes de una empresa hicieron un requerimiento particular, desean tener acceso a un resumen de la información recolectada, desde cualquier plataforma, este resumen incluye tablas y gráficos, este sistema para la gerencia debe estar protegido por contraseña.

Evaluando los requerimientos de la empresa en reunión de jefes de proyecto, se decidió desarrollar un sistema web debido a que este es accesible desde cualquier plataforma. También se vio por conveniente desarrollar está web con una bases y arquitectura que permita a futuro adaptarla a las demás empresas, asumiendo que a futuro los demás clientes tendrán el mismo requerimiento.

El principal problema es que no se cuentan con recursos expertos para hacer este software por completo, así que por su experiencia se le designó realizar las bases y arquitectura del proyecto, luego, pasará el proyecto a los desarrolladores novatos para que lo concluyan.

Se tiene un tiempo estimado de 3 semanas para completar el proyecto en su totalidad, por lo tanto usted tiene 1 semana para completar las bases y arquitectura del sistema web. Las otra semana se designará al equipo de software novato que desarrollen los requerimientos específicos requeridos. y la última semana se designará para pruebas.

Requerimientos de Diseño:

El diseño de la interfaz es parte fundamental del diseño general de una aplicacion web para que estas interfacez es necesario que el diseño se ajuste a las habilidades, experiencia y expectativas de los usuarios finales.

Lo que un diseñador debe tener claro es como interactuara los usuarios en la pagina web y como se mostrara la informacion a los usuarios.

El proceso de diseño de la interfaz de usuario es donde los usuarios deben de interactuar con los diseñadores y los prototipos de la interfaz, para tomar una decisión de las características, apariencia y funcionamiento de la interfaz de usuario del sistema. Las actividades esenciales del proceso son:

Análisis del usuario. Se desarrolla una comprensión de las tareas que el usuario(Gerentes) realiza, su entorno de trabajo, etc.

Prototipo del sistema. El diseño y desarrollo de la interfaz es un proceso iterativo. Se diseñan varios prototipos y se establece el tipo de framework se utilizara y se muestran a los usuarios para que ellos elijan el adecuado y guiar al diseñador en la evolución de la interfaz.

Evaluación de la interfaz. Es la recopilación de las experiencias de los usuarios con la interfaz, aunque ya se haya hablado con ellos en la actividad del prototipo.

dehaze Se usara Frameworks Css?

En este caso se ve necesario no implementar aun un Framework Css primeramente debido a que no es en si una pagina web que muestre contenido para los usuarios, esta aplicacion es para uso exclusivo de Gerentes y se establece que no tiene que ser rico en estilos visuales si no en contenido que pueda precisar.

Por este motivo No sera necesario implementar un framework Css, si no trabajar con estilos basicos que permitan ofrecer mayor rendimiento.

exposure_plus_1 Ventajas de la decision

  • Mejor rendiemiento en la aplicacion por el detalle de tener menos codigo Css.
  • No se descarta en el futuro poder utilizar un Framework seria facil realizar la migracion.
  • No dependemos de ningun tipo de archivos Js para la ejecucion css basico.
  • Al ser Css basico los desarrolladores novatos podran realizar el trabajo sin problemas al no tener una curva de aprendizaje grande.
  • Al trabajar con css basico no se tendria dificultad en la compatibilidad con navegadores.

exposure_neg_1 Desventajas de la decision

  • Corremos el riesgo de no tener un buen marco de trabajo y buenas practicas que nos ofrecen los Frameworks.
  • Corremos el riesgo que talves nuestros codigos se descarten mas adelante.
